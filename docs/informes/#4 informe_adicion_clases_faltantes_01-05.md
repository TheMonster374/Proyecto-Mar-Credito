## 📌 Resumen

Este informe documenta la implementación inicial de las clases base necesarias para estructurar el sistema **Mar Crédito** y permitir el desarrollo posterior de la lógica del proyecto.

---

## ☕ 1. Implementación de clases del dominio

### ✅ Objetivo

Completar las entidades principales necesarias para representar usuarios, préstamos, acuerdos y demás elementos fundamentales del sistema.

---

### 🔧 Actividades realizadas

- Se agregaron y organizaron las clases principales del proyecto.
- Se definieron atributos y métodos básicos para representar operaciones futuras del sistema.
- Se estructuró el proyecto utilizando paquetes separados para mantener el código ordenado.

---

### 📦 Clases implementadas

Ruta principal:

- `src/marcredito/model/`

Clases trabajadas:

- `Usuario`
- `Solicitante`
- `Prestamista`
- `Prestamo`
- `Pago`
- `Acuerdo`
- `PQRS`

---

### 🧩 Resultado

- El proyecto quedó con una base funcional para continuar el desarrollo.
- Se preparó la estructura necesaria para integrar posteriormente lógica de negocio e interfaces gráficas.
