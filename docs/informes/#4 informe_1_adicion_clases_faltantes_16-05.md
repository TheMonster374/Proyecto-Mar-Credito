## 🎯 Objetivo

Completar el conjunto de clases necesarias para que el sistema tuviera sus entidades principales (usuarios, préstamos, pagos, PQRS, etc.) y una base funcional sobre la cual continuar el desarrollo.

---

## 🛠️ Actividades realizadas

- Se agregaron y/o completaron las clases requeridas en el proyecto para representar los elementos del sistema.
- Se definieron atributos y métodos básicos en cada entidad para soportar operaciones futuras (por ejemplo: creación de usuarios, manejo de préstamos y pagos, y registro de PQRS).
- Se organizó el código en paquetes coherentes para mantener el proyecto ordenado y facilitar su mantenimiento.

---

## ✅ Resultado

- El repositorio quedó con las **clases esenciales implementadas**, evitando bloqueos por faltantes.
- Se dejó una base sólida para continuar con la lógica del sistema y, posteriormente, con la integración de la interfaz.

---

## 📎 Evidencias (en repositorio)

- Carpeta: `src/marcredito/model/`
- Clases: `Usuario`, `Solicitante`, `Prestamista`, `Prestamo`, `Pago`, `Acuerdo`, `PQRS`, entre otras.
